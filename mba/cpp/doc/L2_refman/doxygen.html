The Livingstone Reference Manual and doxygen

 

The Livingstone Reference Manual uses the doxygen document-generation system.

doxygen is like Javadoc in that it creates HTML documentation by extracting information from declarations, definitions and comments in source files. Unlike Javadoc, however, doxygen works with C, C++ and IDL source files and can produce several different document types. It also has many more features.

Because the Reference Manual involves hundreds of files and can be reproduced (and customized) by anyone with the source code, these pages are not distributed with the Livingstone source code. An up-to-date version of these pages is served from the Skunkworks web site at http://ic.arc.nasa.gov/projects/mba/projects/L2/doc/L2_refman/index.html . This documentation is based on the *.h files only, and is not cross-indexed to source code.

If you have the Livingstone source code and wish to install the Livingstone Reference Manual at your site, you can generate the HTML pages by extracting them yourself using doxygen. The present document aims to make this process a bit easier.

Installing doxygen

doxygen is open source; you can download and install it from the doxygen web site at http://www.doxygen.org. If you have additional tools in your environment, such as Qt and GraphViz, you can interact with doxygen via a GUI, produce documentation in formats other than HTML (e.g., LaTex, RTF or PDF) and generate more sophisticated graphics, but you can produce useful HTML pages with only the minimal required GNU tools (flex, bison and make).

You can download a binary file for several operating systems. If it is available for your environment, save yourself a lot of trouble and do so.

If you can not download a binary, you will have to build the binary from source. Download the tarball from the doxygen web site, extract the source, and run the configure script.

You will almost certainly have to supply command-line options to configure. In particular,

  • if you are not root, you will have to supply as value for the --prefix option some directory into which the doxygen binary will be written; and
  • you will probably have to specify the platform with the --platform option, selected from those listed in file PLATFORMS, which is part of the distribution.

Inclusion and collaboration graphs, which are included in the on-line Reference Manual, require GraphViz, which you can download and install. Running doxygen from a GUI and producing documentation in other formats require additional software components.

You might want to put the directory containing the executable into your PATH environment variable.

Generating the Documentation

We have supplied a doxygen project file mba/cpp/doc/L2_refman/Doxyfile. The HTML pages it produces suit our purposes, but you might wish to customize this file to make the HTML better suited for your needs. The doxygen developers made sure that the project file is copiously self-documented, and you can get additional documenation about the configuration parameters at the Doxygen web site. After you are satisfied with the contents of this project file, you can create the documentation with the command: 

doxygen Doxyfile
If you are using doxygen's GUI, you can use it to modify the project file and create the documentation.

The supplied project file will place all the generated HTML, including the root frameset page index.html, in directory mba/cpp/doc/L2_refman/html/. Feel free to change this. However, please note that the "main page" of the documentation comes from a comment in file mba/cpp/include/api/livingstone.h. This main page contains HTML links to non-doxygen-generated web pages in directory mba/cpp/doc/L2_refman/ and its siblings. If you do not wish to preserve the default directory structure, you should edit either this C++ header file or the generated HTML page(s) to fix or remove the broken links.
