Building the Livingstone2 engine

Back to Main Page

 

 

 


 Windows:
 --------
 Make sure you have checked out the following modules from CVS
 in the same directory:

  L2           - creates the directory mba/cpp and this file
  xml-windows  - creates the directory support/xerces

  Ensure ..\support\xerces\Build\Win32\VC6\Debug is in your path

  Open mba/mba.dsw in Visual C++.   Build the project called
  "test files"


 Solaris with Sun's cc
 ------
 Make sure you have checked out the following modules from CVS
 in the same directory:

  L2           - creates the directory mba/cpp and this file
  xml-solaris  - creates the directory support/xml/xml4c3_1_0-SolCC


  Before building, you must set the environment variable "CCDIR" equal
  to whever the Sun C++ compiler is installed.   

  At Ames this is "/opt/local/SUN/SUNWspro/bin". 

  This causes the correct version of the Sun compiler and linker to be
  invoked in the makefiles. Other compilers, and even other versions of
  the Sun compiler may not work.

   Next, cd to "mba/cpp/src/test" and execute "make". This should
   build the entire project.

   Just ignore the copious link errors...they have to do with template
   instantiations and are corrected by multiple compile passes (automatically
   executed by the makefile). Header file dependencies are not currently
   tracked. The build is debug only.

   For those interested in building other configurations: Note the source
   for the xml parser is different under Windows and Unix. The Unix source
   tree can be downloaded from www.alphaworks.ibm.com/tech under 
   "XML for C++". I had difficulty building it under Solaris CC, but
   judging from the discussion postings, it's much friendlier to gcc.
